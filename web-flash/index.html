当前浏览器不支持 Web Serial API，请使用 Chrome 或 Edge。

如果无法连接串口，请按住 BOOT，点按 RESET，松开 RESET 后再松开 BOOT，然后重新点击烧录按钮。 固件文件已随 GitHub Pages 同源托管，避免浏览器从 Release 直链下载时出现跨域失败。

1

连接设备

点击烧录按钮，授权浏览器访问开发板串口。

2

烧录 Release 固件

页面会按分区地址写入 bootloader、应用、资源、系统分区和存储分区。

3

连接热点配网

烧录后重启，连接设备热点并打开配网页面。

烧录前建议

如果开发板之前烧录过其他固件，建议在弹窗中选择清空设备或先用 esptool erase_flash 清空 Flash，避免旧 NVS 配置影响首次配网。清空会删除原来的 Wi‑Fi、LLM、设备配置。

Wi‑Fi 配网

烧录完成后按 RESET 重启。首次启动没有保存 Wi‑Fi 时，设备会开启配网页面热点： 

esp-claw-xxxxxx

手机或电脑连接该热点后，浏览器打开：

http://192.168.4.1/

在网页中的「基础设置 / Wi‑Fi 设置」填写路由器 SSID 和密码，保存后重启。连接成功后，串口日志会显示 Wi‑Fi STA ready 和设备 IP，之后可通过 http://设备IP/ 访问配置页面。

离线烧录

如果浏览器烧录不可用，可以从 Release 下载 esp-claw-szpi_esp32s3-firmware.zip，解压后用 esptool 手动写入： 

python -m esptool --chip esp32s3 -p PORT -b 460800 \
  --before default_reset --after hard_reset write_flash \
  --flash_mode dio --flash_size 16MB --flash_freq 80m \
  0x0 bootloader.bin \
  0x9000 partition-table.bin \
  0x10000 ota_data_initial.bin \
  0x20000 edge_agent.bin \
  0x820000 emote.bin \
  0xb20000 system.bin \
  0xc20000 storage.bin
固件来自 Release v0.1.11。
